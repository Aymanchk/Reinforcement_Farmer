## 1. Что это такое

Это **обучающая игра-симулятор фермы**, где агент (ИИ-фермер) учится с нуля
управлять полем 3×4 клетки методом **Reinforcement Learning** (обучение с подкреплением).

Алгоритм обучения — **табличный Q-Learning** (без нейросетей).

Агент сам открывает оптимальную стратегию:
**пахать → сажать → поливать → собирать урожай → не уходить в банкротство.**

---

## 2. Концепция Reinforcement Learning

В RL есть три ключевые сущности:

| Сущность | В нашем проекте |
|---|---|
| **Среда (Environment)** | Ферма с клетками |
| **Агент (Agent)** | Фермер, выбирающий действия |
| **Награда (Reward)** | Деньги/штрафы за действия |

**Цикл обучения:**

```
Состояние → Действие → Награда → Новое состояние
```

И так тысячи раз, пока агент не выучит, что выгодно.

## 4. game.py — Мир и правила

### 4.1. Состояния клетки

Клетка может быть в одном из **6 состояний**:

```
EMPTY → PLOWED → SEEDED → GROWING → READY → WITHERED
пусто  вспахано  засеяно   растёт   созрело  увяло
```

### 4.2. Действия агента (7 штук)

| Код | Действие | Описание |
|---|---|---|
| `A_PLOW` | Вспахать | Превращает EMPTY/WITHERED → PLOWED |
| `A_WHEAT` | Посадить пшеницу | Стоит $5, требует PLOWED |
| `A_CARROT` | Посадить морковь | Стоит $5, требует PLOWED |
| `A_PUMPKIN` | Посадить тыкву | Стоит $5, требует PLOWED |
| `A_WATER` | Полить | Ускоряет рост на 1 шаг |
| `A_HARVEST` | Собрать | Только из READY, даёт деньги |
| `A_MOVE` | Перейти | Маленький штраф |

### 4.3. Культуры

| Культура | Время роста | Цена продажи |
|---|---|---|
| Пшеница | 5 шагов | $10 |
| Морковь | 8 шагов | $20 |
| Тыква | 12 шагов | $40 |

> **Tradeoff:** тыква в 4 раза дороже пшеницы, но растёт в 2.4 раза дольше
> и больше шансов сгнить.

### 4.4. Система наград

```python
R_SUCCESS  = +1    # успешное действие
R_ILLEGAL  = -1    # бессмысленное действие (полить пустую клетку)
R_WITHER   = -5    # урожай сгнил
R_BANKRUPT = -20   # банкротство (конец эпизода)
R_MOVE     = -0.05 # штраф за бесполезное хождение
```

Награда за `HARVEST` = цена культуры ($10/$20/$40).

> Это **сигнал обучения** — агент пытается максимизировать сумму наград.

### 4.5. Главная функция `apply_action`

Применяет действие к клетке:
1. Считает награду по правилам
2. Обновляет деньги
3. Двигает биологию через `farm.tick()` (растения растут)
4. Проверяет банкротство
5. Возвращает `(reward, money, done, info)`

### 4.6. `farm.tick()` — биологический шаг

Каждый шаг:
- `SEEDED` → `GROWING`
- `GROWING` растёт на 1, политые — на 1 быстрее
- `READY` ждёт 40 шагов, потом гниёт → `WITHERED`

## 5. agent.py — Мозг агента (Q-Learning)

### 5.1. Q-таблица — главная структура

```python
self.Q: dict[State, np.ndarray] = defaultdict(
    lambda: np.zeros(NUM_ACTIONS)
)
```

Это словарь:
- **ключ** — состояние (кортеж из 3 чисел)
- **значение** — массив из 7 Q-значений (по одному на действие)

`Q[s][a]` = «насколько хорошо делать действие `a` в состоянии `s`».

### 5.2. Состояние — сжатое представление

```python
(состояние_клетки, полита_ли, корзина_денег)
```

Денег — 4 корзины: `<20`, `<50`, `<100`, `≥100`.
Итого **~48 уникальных состояний**.

> **Почему сжатие?** Если хранить точные деньги/координаты — таблица была бы
> миллионы строк, агент не успел бы обучиться. Сжатие = генерализация.

### 5.3. ε-greedy выбор действия

```python
if random < ε:
    действие = случайное              # ИССЛЕДУЕМ мир (exploration)
else:
    действие = argmax(Q[s])           # ЭКСПЛУАТИРУЕМ знания (exploitation)
```

ε начинается с **1.0** (всегда случайно) и затухает до **0.01**.
Со временем агент всё больше доверяет своей Q-таблице.

### 5.4. Уравнение Беллмана — сердце Q-Learning

```python
target = r + γ · max(Q[s_next])
Q[s][a] += α · (target - Q[s][a])
```

**По-человечески:**
> «Обновляю свою оценку действия в сторону того, что реально получил (`r`),
> плюс лучшая оценка следующего состояния (`γ·maxQ`).»

### 5.5. Гиперпараметры

| Параметр | Значение | Что значит |
|---|---|---|
| `α` (alpha) | 0.25 | Скорость обучения |
| `γ` (gamma) | 0.95 | Дисконт будущих наград |
| `ε_start` | 1.0 | Стартовая случайность |
| `ε_end` | 0.01 | Минимальная случайность |
| `ε_decay` | 0.97 | Скорость убывания ε |

### 5.6. Сохранение модели

Q-таблица сохраняется через `np.save` в `.npy` файл — это и есть «обученная модель».

```python
agent.save("qtable.npy")  # сохранить
agent.load("qtable.npy")  # загрузить
```

## 6. main.py — Цикл обучения

### 6.1. Класс `Trainer`

Владеет фермой, агентом, рендером. Запускает обучение.

### 6.2. `run_episode` — один эпизод (1 «день фермера»)

```
1. Сброс мира (farm.reset, farmer.reset)
2. Цикл по шагам (макс. 120):
   a. Выбираем целевую клетку (cell_cycle — равномерный обход)
   b. s = agent.state(farm)         ← текущее состояние
   c. a = agent.pick(s)             ← выбор действия (ε-greedy)
   d. r, money = apply_action(...)  ← применили действие
   e. s_next = новое состояние
   f. agent.update(s, a, r, s_next) ← ОБУЧЕНИЕ ЗДЕСЬ
3. agent.decay()                    ← уменьшаем ε после эпизода
```

### 6.3. `cell_cycle` — равномерный обход

Бесконечный перемешанный итератор по всем клеткам.
Гарантия: агент видит все клетки одинаково часто.

### 6.4. Ранняя остановка

Если средняя награда за последние **40** эпизодов не растёт **50** эпизодов
подряд (порог `ES_MIN_DELTA=2.0`) → считаем что обучение сошлось, останавливаемся.

### 6.5. CLI-флаги

| Флаг | Описание |
|---|---|
| `--episodes N` | Число эпизодов (default 400) |
| `--max-steps N` | Шагов в эпизоде (default 120) |
| `--seed N` | Сид для воспроизводимости |
| `--no-render` | Headless-режим (быстрее) |
| `--save PATH` | Сохранить Q-таблицу |
| `--load PATH` | Загрузить Q-таблицу |

## 7. view.py — Визуализация

Три компонента:

### 7.1. `Renderer`
Рисует ферму на pygame: клетки, фермера, HUD, всплывающие "+10/-1" над клетками.

### 7.2. `GifRecorder`
Записывает **2 эпизода** в один GIF:
- **Early** — эпизод 1 (агент случайно тыкается)
- **Late** — поздний эпизод (агент уже умеет играть)

Видно прогресс обучения наглядно.

### 7.3. `plot_rewards`
Строит график через matplotlib:
- Синие точки — награда за каждый эпизод
- Зелёная линия — накопленное среднее (learning curve)
- Жёлтая вертикальная линия — момент сходимости

**Как читать график:**
- Линия растёт → агент учится
- Линия плоская → агент сошёлся
- Линия шумная → ещё исследует

## 11. Возможные вопросы и ответы

### Q: Почему таблица, а не нейросеть?
**A:** У нас всего ~48 состояний — таблица помещается в память и обучается быстрее.
Нейросеть (DQN) нужна, когда состояний миллионы (Atari, шахматы).

### Q: Почему ε уменьшается со временем?
**A:** В начале агент ничего не знает — нужно **исследовать** мир случайно.
Потом, когда Q-таблица наполнилась — выгоднее **использовать** накопленные знания.
Это классический tradeoff exploration vs exploitation.

### Q: Что такое γ=0.95?
**A:** Дисконт-фактор будущих наград.
- γ=0 → агент думает только о текущем шаге (близорукий)
- γ→1 → заглядывает далеко вперёд (дальновидный)
- 0.95 — хороший баланс.

### Q: Как понять, что агент обучился?
**A:** По графику: средняя награда растёт и выходит на плато.
Плюс ранняя остановка триггерится автоматически по критерию плато.

### Q: Что хранится в `.npy` файле?
**A:** Q-таблица — словарь `{состояние: массив Q-значений}`. Это «память» агента.

### Q: Чем Q-Learning отличается от других RL-алгоритмов?
**A:** Q-Learning — **off-policy** алгоритм: учится на лучшем возможном действии
(`max Q[s']`), даже если сам бы выбрал другое из-за ε. Это его сильная сторона.

### Q: Почему именно эти награды?
**A:** Награды — это **сигнал**, который формирует поведение.
- `+1` за успешное действие — учим базовым правилам
- `-1` за бессмысленное — отучаем от бесполезных попыток
- `-5` за гниль — учим успевать собирать
- `-20` за банкротство — учим экономить деньги
- `-0.05` за движение — отучаем бессмысленно бродить

## 12. Глоссарий

| Термин | Определение |
|---|---|
| **Reinforcement Learning** | Обучение через взаимодействие со средой и получение наград |
| **Агент** | Сущность, принимающая решения |
| **Среда** | Мир, в котором действует агент |
| **Состояние (State)** | Текущая ситуация в среде |
| **Действие (Action)** | Что агент может сделать |
| **Награда (Reward)** | Числовой сигнал обратной связи |
| **Эпизод** | Одна полная «игра» от старта до конца |
| **Политика (Policy)** | Стратегия выбора действий |
| **Q-функция** | `Q(s,a)` оценивает «качество» действия `a` в состоянии `s` |
| **Q-таблица** | Табличное представление Q-функции |
| **Уравнение Беллмана** | Рекуррентная формула обновления Q |
| **ε-greedy** | Стратегия: с вероятностью ε случайное действие, иначе лучшее |
| **Exploration** | Исследование мира случайными действиями |
| **Exploitation** | Использование накопленных знаний |
| **α (alpha)** | Скорость обучения — насколько быстро меняем оценки |
| **γ (gamma)** | Дисконт-фактор — важность будущих наград |
| **Сходимость** | Состояние, когда обучение перестаёт давать прирост |
| **DQN** | Deep Q-Network — Q-Learning с нейросетью вместо таблицы |
| **On-policy / Off-policy** | Учится ли агент на своих или на оптимальных действиях |
